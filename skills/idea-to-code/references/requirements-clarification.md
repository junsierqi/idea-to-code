# Requirements Clarification

## Purpose

Use this template when the user gives an idea, a rough concept, or an incomplete design instead of a concrete implementation request.

## Clarification Output

Convert the idea into these fields before coding:

- target outcome
- primary user or actor
- main workflow
- success criteria
- explicit non-goals
- dependencies or constraints
- visible outputs or acceptance evidence

## Rule
