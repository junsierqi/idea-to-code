# Final Report Template

## Purpose

Use this structure when summarizing the full journey from the user's initial idea to the delivered result.

## Report Sections

### 1. Target

State:

- the original idea, feature, or design goal
- the final delivered scope

### 2. Milestones

List the major iterations in order:

- what was decided
- what changed
- why that step mattered

### 3. Implementation

Summarize:

- architecture or structural choices
- major files or modules changed
- important tradeoffs

### 4. Verification

Summarize:

- build and test steps run
- flow coverage achieved
- runtime evidence observed
- remaining unverified areas

### 5. Visual Evidence

Include screenshots when:

- the task has UI, layout, rendering, or visual output
- a screenshot tool or captured image artifact is available

If screenshots are unavailable, say that explicitly and provide the next-best evidence such as logs or terminal output.

### 6. Risks And Follow-Up

State:

- remaining risks
- deferred work
- next recommended milestone

## Style

- keep the report chronological and concrete
- prefer evidence over claims
- separate verified facts from assumptions
