# Delivery Loop

## Purpose

Use this checklist when turning an idea into an implemented feature over multiple iterations.

## Loop

1. Clarify the desired outcome
2. Inspect the current implementation
3. Choose the next milestone
4. Implement the milestone
5. Test it
6. Summarize completed work and remaining risk
7. Continue with the next milestone

## Milestone Quality Bar

A good milestone:

- changes a real artifact in the repo
- can be verified locally
- reduces uncertainty for the next step
- does not leave the project in a broken intermediate state

## Typical Outputs

- architecture note
- protocol change
- state model
- server behavior
- client behavior
- test coverage
- acceptance checklist

## Stop Conditions

Pause only when:

- the user changes direction
- an external approval is required
- a dependency or environment capability is unavailable
- additional implementation would be reckless without clarification
